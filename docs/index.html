§ 1 · What it is

One paper per feature. A whole map of your system.

Cluedoc organizes documentation by capability, not by code layout. Whether your repo is one package or twenty, you get the same thing: a navigable tree of feature papers you can start anywhere in and walk outward from.

features › tree

Capability tree

Every feature is a folder with a README.md paper. Big features contain smaller ones — higher-level docs live in higher folders, finer detail lives deeper.

prose › visual

Visual by default

Each paper opens with a diagram, and leans on Mermaid and terminal graphics throughout. Never a wall of prose when a picture is clearer.

links › graph

A citation graph

Papers cite each other under “Related Work.” Land on any paper and follow links to exactly the context you need — like reading a research bibliography.

code › abstract

About the code, never the code

Prose stays abstract and human — no snippets, no symbols, no paths. A sources list in the frontmatter anchors each paper to the files that realize it.

§ 2 · How it works

Progressive, not one big pass.

Cluedoc builds the docs progressively, driven by your code changes — updating the affected papers as features are added, changed, or removed, without being asked.

1

You change code

Cluedoc locates the change and scans both directions — its callers (the larger feature that contains it) and its callees (the collaborators it builds on).

2

It places the change in the tree

It infers what capability the code serves and finds where it belongs — the parent feature above it and the child features below it, up to a paper that already exists or the root.

3

It updates every affected paper

Parent, self, and children alike. A small change touches one leaf; a structural change ripples across several levels — and every cross-link is kept resolving.

4

It guides your reading

Ask how something works and Cluedoc answers, then appends a short Reading Guide — the few papers most worth reading, in a suggested order.

One command.

The skills CLI auto-detects your agent — Claude Code, Codex, Cursor, and many others — and installs Cluedoc for it.
